# aitools 视觉契约

## 方向

Web 整体风格是「现代经典工作台」。界面要像一套长期使用的个人工具：清爽、克制、有一点纸本文具感，但不复古做旧，不做营销页，不做大面积装饰插画。页面以暖白和墨色建立秩序，用深炭棕作为主操作色，少量橄榄绿和古铜金只用于状态、图表和视觉节奏。

Android 端暂沿用既有粉米暖灰映射，避免未重建 APK 时 Web 与手机端交付混在一次变更里。若后续要同步 Android 皮肤，必须同步更新本文件 Android 映射、校验脚本和所有受影响 APK。

这份文件是契约，不是建议。新增或修改 Web/Android 产品时，必须遵守下面的强制规则。

## 产品原则

- 这是自己日常高频使用的工具，不是对外展示的样板产品。功能要简洁、清晰、直接。
- 不为了撑场面添加解释性文案、装饰性统计、无信息含义的视觉元素或低价值功能。
- 每个输入项、按钮、卡片和统计都必须服务真实使用场景；如果用户不会因此更快记录、查找、同步或理解数据，就不要放。
- 表单优先减少犹豫：同类字段合并，默认值来自事项配置，记录时只填当次真正变化的信息。

## 色彩

- 页面底色：`#f3f1ec`，暖白纸面，只负责整体气质。
- 主文本：`#26231f`，墨色。
- 次级文本：`#625b52`，暖灰。
- Web 卡片底色：`#fffefa`，主要内容面保持接近白色。
- Android 卡片底色：`Color.rgb(255, 249, 246)`，保持接近白的暖纸色。
- 弱底色：`#ece6dc`，浅暖灰，用于输入区、弱选中和分组底。
- 线框：`#d8d0c4`，低对比暖灰线。
- 主操作色：`#36312a`，深炭棕，用于主要按钮、选中态、关键导航。
- 暖强调色：`#7c6f5d`，用于卡片顶边、提示和视觉节奏。
- 金棕强调色：`#ad7f34`，用于打卡、图表柱和少量状态提示。
- 橄榄状态色：`#4f6657`，用于成功、同步和完成态。

Android 映射：

- `PAPER = Color.rgb(253, 239, 239)`
- `CARD = Color.rgb(255, 249, 246)`
- `LINE = Color.rgb(218, 208, 194)`
- `ACCENT = Color.rgb(138, 113, 94)`
- `MUTED = Color.rgb(117, 104, 95)`
- `INK = Color.rgb(47, 41, 37)`

## 背景

Web 背景使用低对比暖白纸面，可以有非常轻的纵向明暗变化，但不铺明显渐变、网格、三角块、圆形光斑或高密度花纹。页面内容应靠接近白色的卡片、细边框和少量色条建立层次。

## 组件

- Web 页面必须引用 `/web/app-shell.css` 和 `/web/product-switcher.css`。
- Web 页面必须使用 `.shellbar`、`.shellbrand`、`.shelltitle`、`.shellactions` 作为顶栏结构。
- Web 产品页不得自己定义新的全局顶栏样式，例如裸 `header { ... }`。
- 产品切换必须通过 `/web/product-switcher.js` 和 `data-product-switcher-host` 挂载。
- 产品切换只能作为顶栏导航控件出现，不允许漂浮大按钮。
- 卡片：8px 圆角，1px 暖色边框，轻阴影，顶部可用 4px 彩色细条。
- 主按钮：奶茶棕实底，白字，8px 圆角。
- 次按钮：白底暖色边框，深色字。
- 工具页：以实际工作区为主，避免大面积空白装饰。

## 页面约定

- 管理台：服务状态和产品卡片，偏控制台。
- 日记：雾粉纸面，标签 chip 自然换行，设置按钮保持小而轻。
- 快印、节拍器：APK-only 工具，不提供 Web 产品页；管理台只展示安装包下载和接口状态。
- 工具市场：偏管理工具，首屏优先显示电脑服务连接状态和工具列表，不做应用商店营销页。
- Web 日历：跨产品使用统一结构和视觉。月份操作使用 `.monthbar`，日历网格使用 `.calendar`，星期格使用 `.weekday`，日期格使用 `.day`；选中日期用主操作色实底，有内容的非选中日期只用底部浅金色提示，不使用角标、气泡、彩色点阵或产品自定义日历样式。
- Android APK：每个独立 App 必须有轻量设置页入口，入口应是小型、低干扰的头部控件。业务 App 设置页至少包含关于、数据/同步边界和必要业务偏好；安装、更新和打开工具统一交给工具市场。
- 局域网探测：工具市场统一承担服务发现和 APK 更新入口。客户端不能只信单一广播返回地址，必须至少尝试已保存地址、UDP 返回地址、UDP 响应来源地址和本机私有网段候选；私有网段候选应并发探测，避免串行超时导致误判。服务端发现响应应返回可连接 URL 列表。需要与 Web 端同步数据的业务 App 可以保留同步所需的电脑服务探测，但探测策略必须与工具市场保持同等级能力。

## Android 标题栏与设置契约

- 所有 Android App 顶部都使用同一类轻量标题栏：左侧保留返回位，中间或左中显示当前页面标题，右侧保留设置入口位。
- 返回按钮必须位于标题栏左侧，使用简单图标样式，例如 `‹` 或等价返回图形；不得使用有边框、有底色的按钮样式。
- 设置入口必须位于标题栏右侧，使用简单图标样式，例如齿轮或等价设置图形；不得使用“设置”文字按钮，不得使用有边框、有底色的按钮样式。
- 普通页面进入设置页后，标题栏标题统一显示“设置”；左侧显示无边框返回图标；右侧设置入口隐藏或留空。
- 设置页内容结构统一使用分组标题 + 卡片内容。所有 App 必须包含“关于”分组。
- 有本地草稿、历史、同步、备份、导入导出等数据能力的 App，设置页必须包含“数据管理”分组；没有数据管理能力的工具可以省略该分组，但必须在“关于”或其它说明里明确数据边界。
- APK 安装、更新、打开工具统一由“工具市场”承担。除确实需要与 Web 端同步数据的 App 外，业务 App 不应出现电脑服务探测、APK 下载或更新入口。
- 需要与 Web 端同步数据的 App 可以保留“电脑服务”分组，但该分组只服务于数据同步，不承担 APK 更新。

## 禁止项

- 禁止产品页重复定义核心 token：`--paper`、`--panel`、`--line`、`--accent`，除非是在共享 `app-shell.css` 或按契约同步更新所有产品页。
- 禁止每个产品单独设计顶栏。
- 禁止引入新的主色体系。
- 禁止产品切换器以浮动主按钮形式出现。
- 禁止大面积纯白空框直接堆在背景上，工具区必须是 `.panel` 风格或等价面板结构。

## 校验

修改 Web 样式后运行：

```bash
node scripts/validate-design-contract.js
```

修改 Android UI、样式、标题栏、设置页或新增 Android App 后也必须运行：

```bash
node scripts/validate-design-contract.js
```

校验脚本必须扫描 `android/*-app/app/src/main/java/**/MainActivity.java` 下所有 Android App，至少覆盖：

- Android 核心颜色常量必须和本文件 Android 映射一致。
- 每个 App 必须有设置页和“关于”分组。
- 设置入口必须是轻量图标，不得是“设置”文字按钮；自绘设置图标必须使用共享齿轮形态，不得退化成菜单/汉堡图标。
- “重置记录”“清空记录”“删除全部”等本机数据破坏性动作不得放在首页/主工作区，必须进入设置页的数据管理语境。

修改 Android 后重新构建受影响 APK；修改工具注册、市场展示或共享构建脚本后运行对应 APK 构建或 `scripts/build-all-apks.sh`。

公开 APK 构建脚本必须先执行 `scripts/preflight-android-contract.sh`。`scripts/build-all-apks.sh` 先统一执行一次契约校验，再用 `SKIP_DESIGN_CONTRACT=1` 调用单个 APK 构建脚本，避免重复校验。不要绕过公开构建脚本直接交付 APK。
